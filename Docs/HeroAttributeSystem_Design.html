功能 / 续航 / 机动

属性含义说明
技能急速 AbilityHaste (AH)降低技能冷却不是直接 CDR！换算 CDR = AH/(AH+100)，可线性叠加且无上限
移动速度 MovementSpeed每秒移动单位需同步到 CharacterMovement->MaxWalkSpeed
物理吸血 PhysicalVamp造成物理伤害按比例回血百分比

为什么还要两个"看不见"的属性？ 战斗结算需要一个瞬时通道来承载"这次造成多少伤害/治疗"——即 IncomingDamage / IncomingHealing（Meta 属性，见 2.4）。它们不上面板、不复制，只在一次 GE 执行内存活。

02GAS 核心概念速览

只讲属性系统必须懂的部分，够你看懂后面所有代码。

2.1 五大件

组件角色本工程放哪
UAbilitySystemComponent (ASC)大脑：持有属性、施加效果、激活技能、管理 TagALOLHeroCharacter（每个英雄/单位各一份）
UAttributeSet数据容器：声明全部 FGameplayAttributeData 属性 + 钳制/结算逻辑同 ASC，挂在英雄身上
UGameplayEffect (GE)唯一修改属性的合法途径：瞬时/持续/无限/周期蓝图资产（如 GE_DefaultAttributes）
UGameplayAbility (GA)技能/普攻逻辑后续阶段
FGameplayTag状态标签（眩晕中、无敌、死亡…）贯穿全系统

2.2 ★ BaseValue vs CurrentValue（最重要的心智模型）

每个 FGameplayAttributeData 同时有两个值，决定了"装备加攻击" 和 "升级加攻击" 走完全不同的路径：

BaseValue（基础值）CurrentValue（当前值）
含义永久的"裸"数值= BaseValue + 所有临时修饰器聚合的结果
被谁改Instant GE（瞬时，永久修改）Duration / Infinite GE（临时修饰器，移除即还原）
典型场景升级 +AD、扣血、消耗蓝装备 +50AD（卸下还原）、Buff +20%AD（到时还原）
读取GetXxxAttribute()...GetBaseValue()GetXxx()（访问器默认返回当前值）
直觉记法装备、Buff 这种"会还原"的加成 → 无限/持续 GE 改 CurrentValue；升级、扣血这种"永久生效"的 → 瞬时 GE 改 BaseValue。

2.3 钳制的三个入口

  • PreAttributeChange(Attr, NewValue&) —— CurrentValue 即将变化前。用于显示层钳制（如把当前生命卡在 [0, MaxHealth]）。注意：这里改 NewValue 不会改 BaseValue。
  • PreAttributeBaseChange(Attr, NewValue&) —— BaseValue 即将被 Instant GE 改变前。硬上限要在这里也钳一次。
  • PostGameplayEffectExecute(Data) —— Instant / 周期 GE 执行后。Meta 属性结算、死亡判定、兜底钳制都在这。

2.4 Meta 属性（伤害管线的核心）

伤害不是"直接减血"，而是先打进一个中间属性 IncomingDamage，在 PostGameplayEffectExecute 里统一经过减伤、吸血、死亡判定后再落到 Health，最后清零。好处：所有复杂计算集中一处，技能只管"我造成 X 点伤害"。

2.5 MMC vs ExecCalc（两种自定义计算）

ModMagnitudeCalculation (MMC)ExecutionCalculation (ExecCalc)
输出单个修饰器的一个标量可同时输出多个属性修饰
适合最大生命=f(等级)、冷却=f(技能急速)伤害公式（捕获双方 AD/护甲/穿透/暴击）
本工程用途等级成长、AH→冷却核心伤害结算（Step 7）

03架构设计

3.1 ASC / AttributeSet 放哪？——归属决策 改为：英雄自身

实体是否需要独立的属性/技能切换 / 复用需求ASC 归属复制模式
英雄 Hero是（每个英雄各异）要做英雄切换展示Hero 自身 ✅Mixed（或单机 Full）
小兵 / 野怪 / 防御塔是用完即弃Pawn 自身Minimal
（备选）单一不可切换英雄是强调跨重生持久PlayerStateMixed
结论：属性放英雄身上 每个英雄都是自包含的战斗实体（自带 ASC + 属性 + 技能 + 冷却）。切换英雄时只需把 PlayerState 的指向换到另一个英雄——它的属性早已就绪，无需迁移、无需重算。这也与"小兵/野怪把 ASC 挂自己身上"的做法统一，心智模型一致。
唯一要注意的权衡：死亡时别销毁英雄 ASC 在英雄 Actor 上 → 若英雄死亡时 Destroy()，属性/等级会随之丢失。LoL 风格本就不该销毁英雄：死亡时隐藏 + 禁用碰撞/输入，计时后原地（或回城）重生，ASC 始终存活，属性自然持久。若以后要跨对局持久的元数据（等级、金币、战绩），那部分再单独存到 ALOLPlayerState（它本就为此而生）。

3.3 初始化时序（ASC 在英雄上 · 英雄被 AI 占有）

ASC 既然在英雄身上，就由英雄自己初始化；因为 Owner = Avatar = 英雄，不依赖 PlayerState。用一个幂等的 InitAbilitySystem() 助手，两端各触发一次：

端触发点动作
服务器ALOLHeroCharacter::PossessedBy()
（AIController 占有时触发，权威端）		InitAbilitySystem() → InitAbilityActorInfo(this,this) + 应用默认属性 GE
客户端ALOLHeroCharacter::BeginPlay()
（AIController 不复制，故客户端走 BeginPlay）		InitAbilitySystem()（幂等，已初始化则跳过）
服务器ALOLPlayerState::SetControlledHero()
（GameMode 调用 / 切换时调用）		换 Owner → PC；相机/HUD 改指向新英雄（属性不在此初始化）
为什么不用 PlayerState 的 OnRep 来初始化了属性已随英雄走，英雄在自己的生命周期里就能完成初始化，逻辑更内聚；切换时也不必反复 init。OnRep_ControlledHero 现在只负责客户端表现（相机跟随、HUD 重绑定），不碰 ASC。
生成顺序的坑（用 Mixed 模式时必看） GameMode::SpawnActor<Hero>() 会在 SetControlledHero 之前就跑完英雄的 BeginPlay，此刻英雄 Owner 还没设成 PC。Mixed 复制模式靠"拥有连接"决定 GE 发给谁——所以请在生成时就 SpawnParams.Owner = NewPC;，保证 BeginPlay 前拥有关系已就绪（见 Step 5）。单机 Full 模式无此问题。

3.4 复制策略

  • ASC：SetIsReplicated(true)。复制模式按场景选： 单机/PIE 演示 → Full（最省心） 多人·玩家英雄 → Mixed（配合 SpawnParams.Owner） 纯 AI 小兵/野怪 → Minimal
  • 推荐：英雄类用 Mixed（前向兼容多人）；若现在只做单机展示，Full 亦可，之后再切。
  • 属性：每个用 DOREPLIFETIME_CONDITION_NOTIFY(..., COND_None, REPNOTIFY_Always)。REPNOTIFY_Always 保证即使预测值与服务器值相同也触发 OnRep，避免 UI/预测错位。
  • Meta 属性（IncomingDamage/Healing）不复制。
  • 利好：ASC 现在挂在 Character 上，而 Character 默认 NetUpdateFrequency 本就较高（移动需要）——不必像放 PlayerState 时那样手动调到 100。

3.6 英雄切换流程（本设计的核心收益）

因为每个英雄自带独立的 ASC / 属性 / 技能，切换英雄不需要迁移任何数据——只是改变"谁是当前操控对象"。

英雄 A
自带 ASC/属性（存活）
⇄ 切换 ⇄
PlayerState
ControlledHero 指向
⇄
英雄 B
自带 ASC/属性（存活）
  1. (服务器) 调 PlayerState->SetControlledHero(NewHero)：旧英雄 SetOwner(nullptr)，新英雄 SetOwner(PC)。
  2. ControlledHero 复制 → 客户端 OnRep_ControlledHero（服务器本地直接执行）：相机改为跟随新英雄（复用你已有的锁定/居中逻辑），HUD 解绑旧英雄、绑定新英雄的属性变化委托。
  3. 输入路由：ALOLPlayerController 的技能输入统一走 GetControlledHero()->GetAbilitySystemComponent()->...，天然指向当前英雄，无需改动。
  4. 新英雄的属性早在它自己的 PossessedBy/BeginPlay 就已就绪 → 切换零迁移、零重算。
想"复用同一个英雄实例"切换造型/数据？也可以不换 Actor，而是对同一英雄重新应用一套"英雄定义"（属性曲线表 + 技能集 + 网格/动画）。但换 Actor 更简单干净，推荐展示期就用多英雄实例 + 切指向。

05战斗公式参考

这些是 ExecCalc / MMC 里要实现的数学，全部对齐英雄联盟的真实规则。

护甲 / 魔抗减伤（含穿透）

// 1) 先算百分比穿透，再减数值穿透；穿透不能把正防御压到 0 以下
ReducedArmor = Armor × (1 − ArmorPenPercent)
EffectiveArmor = ReducedArmor − ArmorPenFlat
if (Armor > 0) EffectiveArmor = max(0, EffectiveArmor)

// 2) 减伤系数（负护甲 = 增伤）
若 EffectiveArmor ≥ 0 : Mul = 100 / (100 + EffectiveArmor)
若 EffectiveArmor < 0 : Mul = 2 − 100 / (100 − EffectiveArmor)

最终物理伤害 = 原始物理伤害 × Mul

魔法伤害同理，把 Armor→MagicResistance、ArmorPen*→MagicPen* 即可。

暴击

isCrit = Rand() < CritChance
物理伤害 ×= isCrit ? CritDamage : 1.0   // CritDamage 默认 2.0 (200%)

攻击速度

AttackInterval = 1 / clamp(AttackSpeed, 0.2, 2.5)
攻击动画播放速率 = AttackSpeed / 基础攻速

技能急速 → 冷却

CDR = AbilityHaste / (AbilityHaste + 100)
有效冷却 = 基础冷却 × (1 − CDR) = 基础冷却 × 100 / (100 + AbilityHaste)
// 例：AH=100 → 50% CDR；AH=200 → 66.7%

韧性 → 控制时长

实际CC时长 = 基础时长 × (1 − Tenacity)
多来源乘法叠加：Tenacity_total = 1 − Π(1 − Tᵢ)

吸血 / 治疗护盾 / 回复

回血(物理) = 已造成物理伤害 × PhysicalVamp
回血(法术) = 已造成法术伤害 × MagicVamp
实际治疗/护盾 = 基础值 × (1 + HealShieldPower)  // 作用于施加方
每周期回复 : Health += HealthRegen × Period  (并 clamp 到 MaxHealth)

void ULOLAttributeSet::PreAttributeChange(const FGameplayAttribute& A, float& V)
{

}

    Super::PreAttributeBaseChange(A, V);
    ClampAttribute(A, V);

void ULOLAttributeSet::ClampAttribute(const FGameplayAttribute& A, float& V) const
{
    if (A == GetHealthAttribute())          V = FMath::Clamp(V, 0.f, GetMaxHealth());
    else if (A == GetManaAttribute())      V = FMath::Clamp(V, 0.f, GetMaxMana());
    else if (A == GetAttackSpeedAttribute()) V = FMath::Clamp(V, 0.2f, 2.5f);
    else if (A == GetCritChanceAttribute() || A == GetArmorPenPercentAttribute()
          || A == GetMagicPenPercentAttribute() || A == GetTenacityAttribute())
                                              V = FMath::Clamp(V, 0.f, 1.f);
    else if (A == GetMaxHealthAttribute() || A == GetMaxManaAttribute()
          || A == GetAttackDamageAttribute() || A == GetAbilityPowerAttribute()
          /* …其余"≥0"属性… */)             V = FMath::Max(V, 0.f);
}
④ 结算：Meta 属性 → Health（伤害管线落点）
void ULOLAttributeSet::PostGameplayEffectExecute(const FGameplayEffectModCallbackData& Data)
{
    Super::PostGameplayEffectExecute(Data);
    const FGameplayAttribute& Attr = Data.EvaluatedData.Attribute;

    if (Attr == GetIncomingDamageAttribute())
    {
        const float Dmg = GetIncomingDamage();
        SetIncomingDamage(0.f);                       // 用完即清
        if (Dmg > 0.f)
        {
            SetHealth(FMath::Clamp(GetHealth() - Dmg, 0.f, GetMaxHealth()));
            if (GetHealth() <= 0.f) { /* 广播死亡 / 打 State.Dead 标签 */ }
        }
    }
    else if (Attr == GetIncomingHealingAttribute())
    {
        const float Heal = GetIncomingHealing();
        SetIncomingHealing(0.f);
        SetHealth(FMath::Clamp(GetHealth() + Heal, 0.f, GetMaxHealth()));
    }
    else if (Attr == GetHealthAttribute())
        SetHealth(FMath::Clamp(GetHealth(), 0.f, GetMaxHealth()));
    else if (Attr == GetManaAttribute())
        SetMana(FMath::Clamp(GetMana(), 0.f, GetMaxMana()));
}

ALOLHeroCharacter::ALOLHeroCharacter()
{

    AbilitySystemComponent = CreateDefaultSubobject<ULOLAbilitySystemComponent>(TEXT("ASC"));
    AbilitySystemComponent->SetIsReplicated(true);
    AbilitySystemComponent->SetReplicationMode(EGameplayEffectReplicationMode::Mixed); // 单机可用 Full

}

UAbilitySystemComponent* ALOLHeroCharacter::GetAbilitySystemComponent() const
{ return AbilitySystemComponent; }

void ALOLHeroCharacter::PossessedBy(AController* NewController)
{
    Super::PossessedBy(NewController);
    InitAbilitySystem();   // 服务器（AI 占有时触发）
}

void ALOLHeroCharacter::BeginPlay()
{
    Super::BeginPlay();
    InitAbilitySystem();   // 客户端（幂等；服务器已初始化则跳过）
}

void ALOLHeroCharacter::InitAbilitySystem()
{
    if (bAbilitySystemInitialized || !AbilitySystemComponent || !AttributeSet) return;
    bAbilitySystemInitialized = true;

    AbilitySystemComponent->InitAbilityActorInfo(this, this);   // Owner = Avatar = 英雄自身

    // 移动速度属性变化 → 同步到 CharacterMovement
    AbilitySystemComponent->GetGameplayAttributeValueChangeDelegate(
        AttributeSet->GetMovementSpeedAttribute())
        .AddUObject(this, &ALOLHeroCharacter::OnMovementSpeedChanged);

    if (HasAuthority())
        InitDefaultAttributes();   // 仅服务器；属性值会复制到客户端
}

void ALOLHeroCharacter::InitDefaultAttributes()
{
    if (!DefaultAttributesEffect || !AbilitySystemComponent) return;
    FGameplayEffectContextHandle Ctx = AbilitySystemComponent->MakeEffectContext();
    Ctx.AddSourceObject(this);
    const FGameplayEffectSpecHandle Spec =
        AbilitySystemComponent->MakeOutgoingSpec(DefaultAttributesEffect, 1.f, Ctx);
    if (Spec.IsValid())
        AbilitySystemComponent->ApplyGameplayEffectSpecToSelf(*Spec.Data.Get());
}

void ALOLHeroCharacter::OnMovementSpeedChanged(const FOnAttributeChangeData& Data)
{
    GetCharacterMovement()->MaxWalkSpeed = Data.NewValue;
}
为什么用幂等 helper服务器走 PossessedBy、客户端走 BeginPlay（AIController 不复制，客户端没有 PossessedBy）。用 bAbilitySystemInitialized 保证每台机器只初始化一次，既覆盖两端、又避免重复绑定委托。

Step 6默认属性：用 GE + Curve Table 做等级成长

  1. 在编辑器新建 GameplayEffect 蓝图 GE_DefaultAttributes，Duration Policy = Instant。
  2. 为每个要初始化的属性加一个 Modifier，Op = Override，Magnitude 选 Scalable Float。
  3. 新建 Curve Table CT_HeroBaseStats，每个属性一条曲线，X=英雄等级、Y=数值（如 MaxHealth：1级640，每级+100）。把 Scalable Float 关联到对应曲线。
  4. 在英雄蓝图里把 DefaultAttributesEffect 指给 GE_DefaultAttributes（每个英雄可指向各自的成长表，天然支持"不同英雄不同数值"）。升级时按新等级重应用一次即可刷新成长。

08命名规范 & 常见坑

命名规范
